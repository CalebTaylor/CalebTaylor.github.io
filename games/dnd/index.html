I wrote the netcode and managed the servers for a Dungeons and Dragons app.

The app had a few network components:

  • OAuth with Patreon.
    • Only users who paid through patreon had access to the app.
  • Cloud saves
    • Users needed to be able to access their data anywhere, on any machine, so we implemented the saving of serialized data to the cloud (AWS)
  • DM client to player client webhooks
    • An important aspect of table top combat is fog-of-war. We made it so that the DM could push an updated version of the battle map to players on their phones or laptops.
